Etiquetas de Formularios

<form>

Formulario

Definición: Contenedor que agrupa todos los controles de entrada. Define a dónde se envían los datos (action) y cómo se envían (method: GET o POST).

<form action="/enviar" method="POST">
  <!-- inputs, labels, botones van aquí -->
</form>

El atributo action define la URL a la que se envían los datos. method="POST" los envía en el cuerpo de la petición.

Usa siempre POST para contraseñas o datos personales.

<label>

Etiqueta de campo

Definición: Asocia un texto descriptivo a un control de formulario. Al hacer clic en el label, el campo correspondiente recibe el foco automáticamente.

<label for="nombre">Nombre completo:</label>
<input type="text" id="nombre" name="nombre">

El <label> es fundamental para accesibilidad.

<input>

Campo de entrada

Definición: El elemento más versátil de formularios. Su comportamiento cambia completamente según el atributo type. Es una etiqueta vacía (sin cierre).

<input type="text"     placeholder="Texto libre">
<input type="email"    placeholder="correo@ejemplo.com">
<input type="password" placeholder="Contraseña">
<input type="number"   min="0" max="100">
<input type="date">
<input type="checkbox" name="acepto">
<input type="radio"    name="opcion" value="a">
<textarea>

Área de texto

Definición: Campo de texto de múltiples líneas. El usuario puede redimensionarlo. Se definen las filas visibles con rows.

<textarea rows="4" placeholder="Escribe aquí..."
          maxlength="500"></textarea>
<button>

Botón

Definición: Crea un botón interactivo. Más flexible que <input type="submit"> porque puede contener HTML (iconos, imágenes, texto formateado).

<button type="submit">Enviar formulario</button>
<button type="reset">Limpiar</button>
<button type="button" onclick="miFuncion()">Acción</button>
